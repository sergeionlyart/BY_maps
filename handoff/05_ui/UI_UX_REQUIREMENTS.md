# Требования по UI/UX — BY Maps, релиз 1.1

**Кому:** группа разработки · **Дата:** 12.07.2026 · **Объект:** https://by-population-maps.vercel.app (состояние после релиза 1.0, коммит e30ad92)

Цель: сделать сайт удобным для чтения и восприятия. Приоритет — мобильная версия, где картой сейчас пользоваться практически невозможно. Все проблемы ниже — не мнения, а результаты замеров живого сайта и анализа кода (методика замеров — в конце документа).

## 2. Целевая архитектура: лендинг + полноэкранная карта

Отвечаем на вопрос «может, сделать лендинг»: **да, но не вместо карты, а вместо нынешней главной.** Сейчас «/» — это рабочий инструмент, который на телефоне разваливается на 2200 px вертикальной ленты. Разделяем роли:

**U-01 (P1). Главная «/» → повествовательный лендинг** (mobile-first, обычная прокрутка, без интерактивной карты):
1. Хиро: заголовок-тезис, 1 абзац, две кнопки — **«Открыть карту» → /map** и «Читать статью» → /article.
2. 3–5 сюжетных блоков с ключевыми цифрами и статичной графикой (спарклайны/мини-карты как SVG/изображения, не MapLibre): «пик 1989 → −1,14 млн», «село −65 %», «Минск ×4», «три сценария до 2075».
3. Лента карточек исследований (10 шт.), блок «Проверь сам» (артефакты), блок об авторе, футер.
4. Требование: первый экран лендинга информативен без единого тапа; ни одного горизонтального скролла; LCP < 2,5 с на Moto G-классе.

**U-02 (P1). Карта → «/map», режим приложения.** Разметка без прокрутки страницы: карта занимает всю высоту за вычетом компактной шапки и закреплённого таймбара. Существующие диплинки `/?sel=…` редиректить на `/map?sel=…` (301/js) — ссылки из статей и исследований не должны умереть.

Если решите не заводить /map отдельным маршрутом — допустимая альтернатива: «/» остаётся картой на десктопе, а на мобильных до карты показывается лендинг-часть с кнопкой «К карте» (anchor). Но вариант с /map чище и для аналитики, и для навигации.

## 3. Мобильная карта (/map) — обязательные требования (P0)

**U-03. Совместные жесты.** Включить `cooperativeGestures: true` в MapLibre (двухпальцевый пан, зум щипком; одиночный палец прокручивает страницу там, где она есть). В полноэкранном режиме /map, где прокрутки нет, — оставить одиночный пан, но тогда U-02 обязателен. Критерий: на «/» (пока карта там) страницу можно проскроллить пальцем через карту.

**U-04. Слайдер лет всегда на экране.** Таймбар закрепляется внизу вьюпорта (`position: sticky/fixed`, высота ≤ 64 px + safe-area). Изменение года мгновенно видно на карте без прокрутки. Тач-зона ползунка ≥ 44×44 px (кастомный thumb или увеличенный хит-бокс); деления — только переписные годы, прореженные по ширине (при <480 px показывать 4–5 меток: 1897, 1959, 1989, 2019, 2075); «прогноз» — бейдж у правого края, без переноса.

**U-05. Контролы → компактная панель.** На мобильных над картой остаётся одна строка: селектор уровня (Области/Районы/Города) + кнопка **«Слои и показатель»**, открывающая bottom-sheet со всеми остальными настройками (показатель, весь район/без центра, сценарий, стартовый ряд, города точками, граница 1921, база сравнения). Все интерактивные элементы ≥ 44 px. Выбранные нестандартные настройки показываются чипами над картой.

**U-06. Карточка территории → bottom-sheet.** Тап по району/городу открывает шторку снизу (свёрнутая: название + население + динамика, ~30 % экрана; растягивается до полной карточки с графиком; смахивание вниз закрывает). Вкладки Территория/Сравнение/Урбанизация живут внутри шторки. Критерий: после тапа по полигону обратная связь появляется в пределах видимого экрана за <300 мс.

**U-07. Легенда сворачиваемая.** По умолчанию — компактный чип «Легенда» (или полоска градиента высотой 8 px с подписями min/max); разворачивается тапом поверх карты, закрывается тапом вне. Развёрнутая легенда не должна перекрывать >25 % карты.

**U-08. Тултип → тап.** На тач-устройствах первый тап по полигону показывает мини-тултип (название + значение + тип данных) и подсвечивает объект, открытие шторки — по тултипу или повторному тапу. Иначе пометки достоверности (важная часть методологии) на телефоне не существуют.

**U-09. Стартовый вид.** `fitBounds` с padding, зависящим от вьюпорта (сейчас константа 18): страна целиком, по центру, с запасом под легенду-чип и таймбар. Кнопка «⌖ Вся страна» возвращает вид.

## 4. Мобильная навигация и типографика (P0/P1)

**U-10 (P0). Навигация в одну строку ≤ 56 px.** Вместо переноса шести пунктов на 3 строки: бренд (сокращённый до «BY Maps» или лого) + бургер, открывающий шторку с пунктами и переключателем RU/BY. На контентных страницах — sticky. Критерий: при 360 px навигация занимает одну строку.

**U-11 (P0). Типографика чтения.** Для статьи и контентных страниц: базовый кегль **17 px (mobile) / 18 px (desktop)**, line-height 1,6–1,7, `max-width` текстовой колонки **~68–72ch (≈ 640–700 px)**; h1 28–32 px mobile; отступы между абзацами ≥ 0,9 em. Мелкий кегль (12–13 px) оставить только подписям и сноскам. Критерий: на /article — 45–65 знаков в строке mobile, 60–80 desktop.

**U-12 (P1). Анонс на карте-странице** сократить до одной строки с «подробнее →» (раскрытие), высвобождает ~100 px первого экрана.

**U-13 (P1). Прочее мобильное:** чекбоксы → переключатели ≥ 44 px; `font-size` полей ≥ 16 px (iOS-зум); `viewport-fit=cover` + safe-area для закреплённого таймбара; проверить графики LineChart на ширинах 320–414 (подписи осей не должны накладываться — сейчас на territory-графике тесно).

## 5. Десктоп (P1–P2)

**U-14 (P1).** Статья/контент: применить U-11 (базовый кегль 18 px был бы достаточен, сейчас 14 px), колонку сузить до ~700 px, оглавление — влево от колонки при ≥1280 px.

**U-15 (P2).** UI-кегль контролов и легенды 13 → 14 px; высота кнопок ≥ 32 px; расстояние между группами контролов увеличить (группировка «что показываем / на каком уровне / слои» с разделителями).

**U-16 (P2).** Адаптив 981–1280 px: боковая колонка 420 → `minmax(340px, 380px)` либо схлопывание в таб под картой до 1120 px.

**U-17 (P2).** Тёмная тема: проверить контрастность легенды и оранжевых точек городов на тёмном фоне (WCAG AA, контраст ≥ 3:1 для графики).

## 6. Критерии приёмки релиза 1.1

1. **Сценарий «телефон, одна рука»:** открыть /map, сменить год слайдером, тапнуть район, увидеть карточку, сменить показатель через шторку, вернуться на лендинг — всё без пинцетных жестов, случайных панов карты и горизонтального скролла. Проверить на реальных iPhone (Safari) и Android (Chrome), ширины 360/390/414.
2. Слайдер и навигация видимы одновременно с картой при 360×640.
3. Lighthouse mobile на «/», /map, /article: Performance ≥ 85, Accessibility ≥ 95; тач-цели проходят аудит «tap targets».
4. Все существующие URL работают (редирект `/?sel=` → `/map?sel=`); RU/BE-паритет новых элементов UI (лендинг, шторки, подписи).
5. Ничего из функциональности не удалено: все показатели, уровни, сценарии, стартовые ряды, сравнение, урбанизация доступны на мобильном (глубже в шторках — допустимо; недоступно — нет).
6. Скриншоты «до/после» для 390 px приложить к PR.

## 7. Приоритеты и оценка

- **P0 (карта становится пригодной):** U-03, U-04, U-05, U-06, U-07, U-08, U-10, U-11 — ориентировочно 3–5 дней.
- **P1 (структура и лендинг):** U-01, U-02, U-09, U-12, U-13, U-14 — ещё 3–4 дня.
- **P2 (полировка):** U-15…U-17.

Рекомендуемый порядок: U-03/U-04 (двухчасовые правки, снимают главную боль) → U-10/U-11 → U-05…U-08 → лендинг U-01/U-02.
