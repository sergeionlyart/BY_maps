# Пакет передачи: раздел «Пирамида» (INF-11)

По результатам исследования темы R-1 («Пирамида, которая переворачивается»). Всё выверено запуском на данных репозитория; готово к разработке.

```
08_pyramid/
├── README.md                  ← этот файл
├── RESEARCH_NOTE.md           результаты исследования: инвентаризация данных,
│                              выверенные факты (шрам войны, эхо, провал 90-х,
│                              обвал 2020-х, перекос полов, «гриб»-2075), риски
├── TZ_PYRAMID.md              ТЗ: задачи P-1…P-12 (ETL, экспорт CCMPP, страница
│                              /pyramid с «найди себя», пакет артефактов, рилс),
│                              приёмка, этапы (~2 недели)
├── build_pyramids_draft.py    рабочий скрипт сборки датасета (stdlib, детерминирован,
│                              встроенные проверки «до человека»)
├── data/pyramids_draft.json   черновой датасет: 39 серий (2009, 2019–2026,
│                              2030–2075 × 3 сценария-заглушки WPP) — для
│                              прототипирования UI уже сейчас
└── content/
    ├── pyramid.ru.md          тексты страницы: интро, аннотации A1–A7, методблок
    └── pyramid.be.md          то же по-белорусски
```

**Ключевые решения, принятые в исследовании (см. RESEARCH_NOTE):**
1. История 1959–1999 берётся из возрастных таблиц переписей (Демоскоп/Белстат) — вопреки исходной гипотезе, в завендоренном HMD населения по возрастам нет (там таблицы смертности).
2. Будущее — только из экспорта CCMPP (задача P-3, модель уже считает структуры); WPP-заглушка прототипа в прод не выходит.
3. Каждый кадр несёт тип достоверности: перепись / оценка / интерполяция / модель — в стиле сайта.

**Порядок работ:** прототип UI можно начинать сразу на `data/pyramids_draft.json`; параллельно E1 (история) и E2 (экспорт CCMPP) по ТЗ.
