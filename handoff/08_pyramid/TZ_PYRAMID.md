## 1. Данные (ETL)

**P-1. pyramids.json — целевой набор.** Формат — как в `data/pyramids_draft.json`:
```
{ version, unit, age_groups[17],
  series: { "<year>" | "<year>:<scenario>": {
      type: census|estimate|un-estimate|interpolated|model,
      source: str, m:[17], f:[17], unknown?: int } },
  annotations: [...] (см. §3) }
```
Ключи будущего — `2030:base … 2075:negative` (+ вариант jumpoff: `…:adjusted` при включении U-переключателя, см. P-3). Инварианты-тесты: суммы серий census/estimate равны итогам переписей/оценок **до человека** (2009 = 9 503 807; 2019 = 9 413 446; 2026 = 9 056 080); `unknown` не теряется; порядок групп фиксирован.

**P-2. История 1959–1999.** Завендорить возрастные таблицы переписей 1959/1970/1979/1989 (Демоскоп Weekly) и 1999 (Белстат/Демоскоп): республика, пол × 5-летние группы. `registry.csv`: URL + даты + условия. Усечённые верхние группы (70+/80+) — поддержать в схеме (UI рисует усечённый верх с пометкой). Опция (P-2b, после релиза): годовой трек WPP 1950–2023 для плавных межпереписных кадров с типом `un-estimate`.

**P-3. Экспорт возрастной детали прогноза.** В `etl/forecast/run.py` добавить выгрузку структур {пол × группа} на узлы 2030–2075 (шаг 5 лет) для трёх сценариев × двух стартовых рядов, уровень BY. Модель уже гоняет эти структуры через `ccmpp.py` — задача только записать. Гейт: сумма экспортированной структуры = итогу сценария в `forecast_v2026_4.csv` (допуск ±0,1%). **Заглушка WPP из прототипа в прод не выходит** — она остаётся в `checks/` пакета как кросс-чек (расхождение WPP-medium vs base на 2075 задокументировать).

**P-4. Межопорная интерполяция.** Кадры между опорными годами истории — линейная интерполяция когорт с типом `interpolated` (в тултипе и легенде честно помечено; паттерн достоверности уже принят на сайте).

## 2. Страница /pyramid

**P-5. Визуализация.** Классическая пирамида: мужчины слева (синий), женщины справа (медный `--accent-2`); ось — 5-летние группы; анимированный морфинг между кадрами (400–600 мс, easing), бары перетекают, а не перерисовываются. Слайдер лет 1959–2075 в паттерне /map (переписные годы — метки; прогнозная зона — штриховка + переключатель сценария base/optimistic/negative и стартового ряда official/adjusted). После 2026 рядом с пирамидой — до трёх полупрозрачных контуров-«призраков» других сценариев (цвета A-3 из `06_brand_ui`).

**P-6. «Найди себя».** Поле «год рождения» (+ пол опционально): когорта подсвечивается и **едет вверх** по пирамиде при движении слайдера; подпись у маркера: «2050 · вам 54 · ваша когорта: N тыс. (сценарий базовый)». За пределами прожитых лет маркер гаснет («вы ещё не родились» / для прошлого — когорта не существует). Deep-link: `?born=1985&year=2050&scenario=base`.

**P-7. Аннотации-события.** По мере движения слайдера всплывают каллауты (тексты — §3): шрам войны, эхо, провал 90-х, обвал 2020-х, перекос полов, «гриб» 2075. На мобильном — одна аннотация за раз, снизу. Кнопка «▶ рассказ» проигрывает весь таймлайн с аннотациями (это же — сцена рилса).

**P-8. Мобильный и качество.** Паттерны релиза 1.1: слайдер закреплён снизу, тач-цели ≥44 px, морфинг 60 fps (транзишены транслейтом/скейлом, без перерисовки SVG на каждый кадр — путь: анимировать width/x баров через CSS transform или canvas). Lighthouse mobile не ниже текущего уровня сайта. RU/BE паритет.

**P-9. Тултипы и достоверность.** Ховер/тап по бару: группа, численность м/ж, f/m, доля группы; тип кадра (перепись/оценка/интерполяция/модель) — бейджем, как всюду на сайте.

## 6. Приёмка

1. Тесты P-1 зелёные (суммы до человека); CCMPP-экспорт согласован с итогами прогноза (±0,1%); в проде нет WPP-заглушки.
2. История: ≥5 переписных кадров 1959–1999 с registry-провенансом; усечённые группы корректно отображаются.
3. «Найди себя» работает на границах (1890-е, 2075), deep-links восстанавливают состояние, RU/BE.
4. Морфинг 60 fps на среднем смартфоне; аннотации не перекрывают пирамиду на 360 px.
5. Пакет v1.0.0 воспроизводим байт-в-байт, AGENT-задания прогнаны собственным агентом до передачи.
6. Скриншоты до/после + мастер рилса RU/BE.
