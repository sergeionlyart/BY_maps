# Косметика и личный бренд — BY Maps, релиз 1.2

**Кому:** группа разработки · **Дата:** 12.07.2026 · **Состояние:** после релиза 1.1 (коммит 4aa1621) — UX приемлемый, задача этого этапа: цветовые акценты + мягкое позиционирование автора.

## Диагноз текущего вида (по живому сайту)

**Цвет.** Вся палитра — тёплые серые поверхности + один синий акцент (`--accent: #2a78d6` / `#3987e5`), которым покрашено всё сразу: ссылки, кнопки, спарклайны, бары, версии пакетов. Карточки лендинга, исследований и разделов визуально неразличимы. Единственные «живые» цвета — оранжевые точки городов на карте и зелёная линия городского населения в графиках — в систему не оформлены. Итог закономерен: «серенько».

**Бренд.** Автор на сайте фактически невидим: на лендинге блок «О проекте» даже не называет имени; /author — стена серого текста без фото, акцентов и call-to-action; в статье — только строка байлайна. Контакты есть, позиционирования нет.

Принцип правок: **хром остаётся нейтральным, цвет получает смысл**. Никакой радуги: (1) данные/инструмент = синий, (2) человек/автор = тёплый медный, (3) семантика сценариев и динамики = зелёный/терракотовый. Это косметика, не редизайн: только токены, рамки, чипы и один новый компонент.

## Часть A. Цветовые акценты

### A-1 (P0). Токены палитры

Добавить в `globals.css` (значения прогнать через контраст-чекер, ориентиры ниже дают AA на соответствующих фонах):

```css
:root {
  --accent-2: #9a5220;        /* медный: автор, человеческий слой */
  --pos: #2e8f4e;             /* рост / оптимистический сценарий */
  --neg: #c4554a;             /* убыль / негативный сценарий */
  --viz-urban: #2e8f4e;       /* формализовать существующий зелёный графиков */
  --chip-data: #2a78d6; --chip-calc: #0f8b8d;
  --chip-model: #7a5cc5; --chip-interp: #a8701d;
}
@media (prefers-color-scheme: dark) {
  :root {
    --accent-2: #d78f57;
    --pos: #4dbd6e;  --neg: #e07a68;
    --viz-urban: #4dbd6e;
    --chip-data: #5b9fe8; --chip-calc: #3fb3b5;
    --chip-model: #a48ae0; --chip-interp: #d9a04a;
  }
}
```

Медный сознательно взят из уже существующих на сайте оранжевых точек городов и `--hist-border` — палитра станет цельной, а не новой.

### A-2 (P0). Лендинг: смысловые акценты карточек

- Стат-карточки «Что показывают данные»: цветная верхняя кромка 3 px + цвет ключевой цифры по смыслу: «−1,14 млн» и «Село −65 %» — `--neg`; «Минск ×4» — `--accent`; «3 сценария» — трёхцветный мини-спарклайн (см. A-3). Текст остаётся нейтральным.
- Карточки исследований: тонкая левая кромка 2 px `--accent` при hover + код «INF-0x» в цвете `--accent` (сейчас серый). Не раскрашивать все карточки по-разному.
- Секция «Проверь сам» — единственная залитая секция страницы: фон `color-mix(in srgb, var(--accent) 6%, var(--page))`, кнопка остаётся синей залитой.
- Хиро: подложка — едва заметный контур Беларуси (SVG, opacity 0.04–0.06, из имеющегося adm0-полигона); первичная кнопка залитая синяя, вторичная — аутлайн (уже так); над заголовком строка-киккер «Открытое исследование · 1897–2075» в `--accent-2`, 13 px, letter-spacing.

### A-3 (P0). Семантика сценариев везде одинакова

Оптимистический = `--pos`, базовый = `--accent`, негативный = `--neg`, плюс различие штрихом (сплошная/длинный штрих/точки) для дальтоников. Применить: спарклайн «3 сценария» на лендинге (сейчас весь синий), переключатель сценариев на /map (активная кнопка подкрашивается цветом сценария), линии и веер прогноза в карточке территории и на страницах прогноза. Легенду карты (choropleth-шкалы) не трогать — это кодирование данных.

### A-4 (P1). Плашки [Данные]/[Расчёт]/[Модель]/[Интерпретация]

Сейчас `.chip` одинаково-серые. Сделать четыре варианта: текст и рамка цветом `--chip-*`, фон `color-mix(… 10%, transparent)`, слева точка-маркер 6 px. Это оживит статью и методологию ровно там, где цвет несёт смысл. В тёмной теме — светлые варианты токенов.

### A-5 (P1). Мелочи

- «пакет v1.0.x» в карточках — нейтральный серый (сейчас синий конкурирует со ссылками «открыть →»).
- Дельты роста/убыли в карточке территории и таблицах перевести на `--pos`/`--neg` (сейчас локальные хексы — унифицировать).
- Hover карточек: подъём тени + кромка, 120 мс transition.
- Проверить AA: синие ссылки на светлом фоне (#2a78d6 на #f9f9f7 ≈ 4,6:1 — на грани, при уменьшении кегля использовать #2266bd).

### Чего не делать

Не красить фоны секций в насыщенные цвета; не делать разноцветную навигацию; не менять choropleth-шкалы карты; не вводить больше двух акцентных семейств (синий + медный); графики населения оставить сине-нейтральными.

## Часть B. Личный бренд (мягкая интеграция)

Позиционирование: **автор — не «владелец сайта», а демонстрация метода.** Тезис «один инженер + LLM-агенты = проверяемое исследование» — это и есть рекламное сообщение; продаём не услуги, а подтверждённую компетенцию. Тон — приглашение, не продажа. Жёсткое правило: внутри текстов исследований и методологии рекламных вставок нет; CTA живут в подвалах страниц, на лендинге и на /author. Всё — в RU и BE версиях (тексты ниже даны парами).

### B-1 (P0). Компонент AuthorCard (byline)

Переиспользуемая карточка: аватар (фото или инициалы «СА» на медном фоне), имя, одна строка титула, 1–2 кнопки. Акцент — `--accent-2` (медная кромка слева 3 px). Размещение:
- конец статьи /article (полный вариант с CTA-абзацем);
- конец каждой research-страницы (компактный вариант: строка + «Об авторе →»);
- лендинг, блок «О проекте» (см. B-2).

Тексты (компактный вариант):
- RU: **Сергей Авдейчик** — к.т.н., AI/ML-инженер. Проект сделан связкой OSINT + LLM-агенты + проверяемые артефакты. [Об авторе →] [Написать]
- BE: **Сяргей Аўдзейчык** — к.т.н., AI/ML-інжынер. Праект зроблены звязкай OSINT + LLM-агенты + правяральныя артэфакты. [Пра аўтара →] [Напісаць]

CTA-абзац для конца статьи:
- RU: «Понравился подход? Я помогаю командам и специалистам строить такие же конвейеры: LLM-агенты, RAG, автоматизация аналитики и профессиональных процессов. Пишите — обсудим вашу задачу, или приходите на менторинг.»
- BE: «Спадабаўся падыход? Я дапамагаю камандам і спецыялістам будаваць такія ж канвееры: LLM-агенты, RAG, аўтаматызацыя аналітыкі і прафесійных працэсаў. Пішыце — абмяркуем вашу задачу, або прыходзьце на ментарынг.»

### B-2 (P0). Лендинг: блок «О проекте» → «Кто это сделал и зачем»

Два предложения о проекте (текущие) + AuthorCard с фото + третья кнопка LinkedIn. Заголовок блока:
- RU: «Кто это сделал и зачем» · BE: «Хто гэта зрабіў і навошта».
Дополнительная строка под карточкой:
- RU: «BY Maps — витрина метода: так же можно разобрать ваши данные и процессы.»
- BE: «BY Maps — вітрына метаду: гэтак жа можна разабраць вашы даныя і працэсы.»

### B-3 (P0). Страница /author: из стены текста — в посадочную

1. Хиро-карточка: аватар 96 px, имя (28 px), титул, чипы компетенций (LLM и агентные системы · RAG · LegalTech · EduTech · CV · Edge AI · Python) — чипы в стиле A-4, нейтральные с медной рамкой.
2. Ряд кнопок-контактов ≥44 px: Email (залитая медная), LinkedIn, GitHub, Medium, Facebook (аутлайн). Не список ссылок, а кнопки.
3. **Новая секция «Чем могу быть полезен» — три карточки** (это ядро лид-магнита):
   - **Менторинг** · RU: «Вход в AI-инженерию и работу с LLM: персональный план, разбор ваших проектов, код-ревью, подготовка к собеседованиям.» · BE: «Уваход у AI-інжынерыю і працу з LLM: персанальны план, разбор вашых праектаў, код-рэўю, падрыхтоўка да сумоўяў.» · CTA: mailto с subject «BY Maps → менторинг».
   - **Разработка** · RU: «AI-копилоты, RAG-системы, агентные конвейеры и автоматизация профессиональных процессов — от прототипа до продакшена.» · BE: «AI-капілоты, RAG-сістэмы, агентныя канвееры і аўтаматызацыя прафесійных працэсаў — ад прататыпа да прадакшэна.» · CTA: subject «BY Maps → разработка».
   - **Исследования и аналитика** · RU: «Data-driven исследования под ключ по модели BY Maps: открытые источники, LLM-агенты, проверяемые артефакты.» · BE: «Data-driven даследаванні пад ключ па мадэлі BY Maps: адкрытыя крыніцы, LLM-агенты, правяральныя артэфакты.» · CTA: subject «BY Maps → исследование».
4. Существующий текст (биография, направления, приглашение к дискуссии) — ниже, без изменений.

### B-4 (P1). Футер всех страниц

Одна строка перед копирайтом:
- RU: «Сделал [Сергей Авдейчик](/author) — AI/ML-инженер · менторинг и разработка → [написать](mailto:…)»
- BE: «Зрабіў [Сяргей Аўдзейчык](/be/author) — AI/ML-інжынер · ментарынг і распрацоўка → [напісаць](mailto:…)»
Ссылка на имя — цветом `--accent-2` (единственное медное в футере).

### B-5 (P1). «Как это сделано» — один мягкий якорь на инструментальных страницах

На /methodology (раздел про LLM) и на /artifacts внизу — один callout-блок в стиле AuthorCard:
- RU: «Весь конвейер проекта — ETL, модели, тесты, пакеты — построен с LLM-агентами. Хотите такой же для ваших данных? [Обсудить →]»
- BE: «Увесь канвеер праекта — ETL, мадэлі, тэсты, пакеты — пабудаваны з LLM-агентамі. Хочаце такі ж для вашых даных? [Абмеркаваць →]»
Не более одного на страницу; внутри research-текстов — никогда.

### B-6 (P1). SEO/шаринг под личный бренд

- JSON-LD `Person` на /author (name, jobTitle, sameAs: LinkedIn/GitHub/Medium/Facebook) + `author` в метаданных статьи и исследований.
- Title /author → «Сергей Авдейчик — AI/ML-инженер, к.т.н. · автор BY Maps».
- og:image: сгенерировать 1200×630 (контур Беларуси + заголовок + строка «Сергей Авдейчик · AI/ML Engineer») для /, /article, /author — сейчас шаринг-карточек нет, а именно их видят в LinkedIn.
- Все mailto — с предзаполненным subject (см. B-3), чтобы входящие лиды самоклассифицировались.

### B-7 (P2). Опционально

Лёгкая privacy-friendly аналитика (Plausible/Umami, без cookies) — увидеть, доходят ли читатели до /author и CTA; страница-«визитка» /author как отдельная og-ссылка для соцсетей; PDF-версия статьи с байлайном.

## Критерии приёмки

1. Скриншоты до/после (лендинг, /map с переключателем сценариев, /article с плашками, /author) в светлой и тёмной темах; все новые цвета проходят WCAG AA (текст ≥4,5:1, крупный/графика ≥3:1) — приложить отчёт контраст-чекера.
2. Сценарии окрашены одинаково на лендинге, /map и в карточке территории; различимы при симуляции дейтеранопии (штрих/точки).
3. Имя автора встречается на пути читателя минимум трижды без скролла-охоты: лендинг-блок, конец статьи, футер; /author содержит секцию «Чем могу быть полезен» с тремя рабочими mailto-CTA (проверить subject).
4. Ни одного рекламного элемента внутри текстов исследований/методблоков; на страницу — не более одного brand-callout.
5. RU/BE паритет всех новых текстов (тексты выше — готовые, содержательно не менять; вычитка BE носителем).
6. Регресс-чек: choropleth-шкалы карты, легенда и плашки достоверности не изменились.

**Приоритеты:** P0 = A-1…A-3 + B-1…B-3 (≈2–3 дня). P1 = A-4, A-5, B-4…B-6 (≈1–2 дня). P2 = B-7.

*Замечание для автора: для B-2/B-3 нужна фотография (квадрат ≥512 px) — либо согласовать аватар-инициалы.*
