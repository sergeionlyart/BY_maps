# Процедура воспроизведения ключевых результатов

Проверено на чистом Linux-окружении (Ubuntu 22, Python 3.10, Node 20) 12.07.2026. Все команды выполняются из корня репозитория, кроме сценария A (не требует репозитория вообще).

## A. Проверка одного исследования без репозитория (15–30 минут)

Достаточно скачанного zip-пакета:

```bash
sha256sum by-maps-zipf-v1.0.0.zip        # 1. целостность (сверить с catalog.json)
unzip by-maps-zipf-v1.0.0.zip -d zipf && cd zipf
cat README.md LIMITATIONS.md             # 2. что утверждается и чего НЕ утверждается
sha256sum -c checks/checksums.sha256     # 3. целостность каждого файла
./run.sh                                 # 4. полный расчёт от сырых данных до финальных файлов
                                         #    со сверкой expected_results в допусках манифеста
```

Критерий успеха: `run.sh` завершается без ошибок и сообщает о совпадении результатов с эталоном. Финальные файлы в `data/final/` в точности соответствуют опубликованным на сайте.

**Вариант для LLM-агента:** передайте архив своей агентной системе с инструкцией «выполни задания из AGENT.md». Три задания: проверка источников по `sources/registry.csv`; воспроизведение расчёта; стресс-тест 3–7 допущений из `params/assumptions.yaml`.

## B. Полное воспроизведение из репозитория

```bash
git clone https://github.com/sergeionlyart/BY_maps && cd BY_maps

# 1. Окружение (см. T-05: в README добавить pyyaml)
python3 -m venv .venv
.venv/bin/pip install shapely pytest pyyaml

# 2. ETL: сырые данные завендорены, сеть не нужна
.venv/bin/python -m etl.fetch          # проверка/докачка сырых данных
.venv/bin/python -m etl.build          # сборка web/public/data (data.json, geo/)

# 3. Инварианты данных: 155 тестов
.venv/bin/python -m pytest etl/tests/ -q
# ожидаемо: 155 passed (≈1,5–2 мин)

# 4. Байт-воспроизводимость всех пакетов (CI-гейт)
.venv/bin/python -m etl.artifacts.build --all --check
# ожидаемо: 11 строк «OK --check: … воспроизводится байт-в-байт»

# 5. Полная валидация пакетов (прогон run.sh каждого + сверка допусков)
.venv/bin/python -m etl.artifacts.validate --all
```

## Известные ограничения воспроизведения

- Шаг `etl.fetch --force` (перезагрузка сырых данных из сети) зависит от доступности внешних источников; для воспроизведения он не нужен — данные завендорены.
- Пакет INF-08 (nightlights) для *пересоздания* сырых экстрактов требует `rasterio`/`numpy` и исходные растры VIIRS (~ГБ); проверка `run.sh` работает на завендоренных экстрактах без этого.
- Полный прогон `etl.artifacts.validate --all` занимает заметное время (прогон run.sh всех 11 пакетов).
- До закрытия T-01 (публичный GitHub) сценарий B доступен только обладателям доступа к репозиторию; сценарий A работает уже сейчас для любого скачавшего пакет.
