# Структура сайта BY Maps — от лендинга к исследовательскому сайту

Рекомендации для группы разработки. Готовый контент всех новых страниц лежит в `../03_content/ru/` и `../03_content/be/` — задача разработки сводится к созданию маршрутов и рендерингу markdown (паттерн уже есть в проекте).

## 1. Карта разделов (целевая)

```
/                    Карта (существует, без изменений)
/research            Исследования (существует)
/research/<slug>     10 исследований (существуют)
/article             ★ Большая обзорная статья               ← article.md
/about               ★ О проекте (идея, характер, почему Беларусь) ← about.md
/goals               ★ Цели и задачи                          ← goals.md
/methodology         Методология — РАСШИРИТЬ                  ← methodology.md (новая обзорная
                     страница; существующие «Методика обработки» и «Источники» остаются
                     вкладками/подразделами на ней)
/artifacts           Данные и артефакты — РАСШИРИТЬ           ← data-artifacts.md (вводный
                     контент) + каталог из ../04_artifacts/ARTIFACTS_CATALOG.md
                     + процедура из ../04_artifacts/REPRODUCTION.md
/artifacts/<slug>    страницы версий пакетов (существуют)
/author              ★ Об авторе                              ← author.md
/be/...              ★ белорусские версии контентных страниц (см. §3)
```

★ — новые маршруты. Файлы контента кладутся в `web/public/content/ru/` и `web/public/content/be/` (существующие `content/*.md` переносятся в `ru/` или остаются для обратной совместимости — на усмотрение команды).

## 2. Навигация

Верхнее меню (сейчас: Карта · Исследования · Методика · Артефакты):

**Карта · Исследования · Статья · Методология · Данные и артефакты · О проекте**

- «О проекте» в меню ведёт на /about; страницы /goals и /author связываются с /about взаимными ссылками (и из футера).
- Футер на всех страницах дополнить: «О проекте · Цели · Автор · GitHub · Лицензия · RU/BE».
- На главной (карте) добавить один абзац-тизер со ссылками на /article и /about — карта перестаёт быть «лендингом без контекста».

## 3. Двуязычие RU/BE (решение заказчика: русский + белорусский)

Минимально инвазивная схема для статического экспорта Next.js:

1. **Контентные страницы** (article, about, goals, methodology-обзор, data-artifacts-вступление, author): параллельные маршруты с префиксом `/be/` (`/be/article`, `/be/about`, …). Рендер — тот же компонент, другой markdown-файл (`content/be/<name>.md`). Все файлы готовы в `../03_content/be/`.
2. **Переключатель языка** в шапке (RU | BY): на контентных страницах — ссылка на параллельный маршрут; на карте и исследованиях (пока одноязычных) — вести на русскую версию и помечать «па-беларуску: хутка».
3. **SEO:** `<html lang>` по маршруту; `hreflang="ru"`/`"be"` взаимные ссылки; отдельные title/description (в шапке каждого md-файла контента даны title и description для метаданных).
4. **Очередь на будущее (P2):** интерфейс карты по-белорусски — задел уже есть: в `data.json` у каждой территории есть поле `be` (белорусское название); исследования переводить по одному, начиная с самых читаемых.

## 4. Требования к реализации новых страниц

- **Статический рендер:** markdown читать на этапе сборки (серверный компонент / `fs.readFileSync` в build), как на страницах исследований, — НЕ клиентским fetch (см. T-08 в FIX_TASKS: /methodology сейчас единственная «пустая» в HTML страница; новые страницы не должны повторить эту ошибку).
- **Компонент:** переиспользовать существующий `components/Markdown.tsx`.
- **Метаданные:** каждому маршруту — `export const metadata` (title/description из шапки md-файла).
- **Разделение утверждений:** в контенте используются пометки-плашки «данные / расчёт / модель / гипотеза» (в markdown — цитаты с префиксами **[Данные]**, **[Расчёт]**, **[Модель]**, **[Интерпретация]**). Желательно оформить их CSS-стилями (4 цветные рамки) — селектор по началу строки или мини-расширение Markdown-компонента.
- **Статья:** длинная (~30 тыс. знаков); добавить оглавление-якоря по h2 (в md уже есть якорные заголовки) и прогресс чтения не нужен.
- **Карточка автора** (/author): фото опционально; контакты — просто ссылки, без формы.
- **Служебные якоря:** на /artifacts обязателен блок с `id="reproduction"` (на него ссылается контент data-artifacts); при добавлении вкладок на /methodology — якоря для «Методики обработки» и «Источников».
- **Внутренние ссылки BE-файлов** уже ведут на `/be/…`-маршруты (для шести контентных страниц) и на общие RU-маршруты для непереведённого (/research/*, /, /data/*) — сохранить эту схему при вёрстке.
