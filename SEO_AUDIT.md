# SEO-аудит: задачи для команды разработки

**Дата проверки:** 13 июля 2026 года
**Сайт:** <https://by-population-maps.vercel.app>
**Цель:** устранить технические барьеры индексации и улучшить видимость русской и белорусской версий в поиске.

## Краткий вывод

Сайт отдаёт содержательные страницы в серверном HTML, имеет понятную навигацию и качественные исследовательские материалы. Это хорошая основа для поисковой индексации. Однако сейчас поисковые роботы не получают два основных механизма обнаружения страниц — `robots.txt` и XML sitemap; также не заданы canonical URL и непоследовательно размечены языковые версии.

Просим команду разработки выполнить задачи в порядке приоритета ниже.

## P0 — блокирует эффективное обнаружение страниц

### 1. Добавить `robots.txt`

**Проблема.** URL <https://by-population-maps.vercel.app/robots.txt> возвращает `404` и HTML-страницу ошибки.

**Риск.** Поисковые системы не получают явных правил обхода и ссылки на sitemap.

**Просьба к команде.** Создать `web/app/robots.ts` или эквивалентный статический файл. Разрешить обход публичных страниц и указать абсолютный URL sitemap.

**Критерии приёмки.**

```text
GET /robots.txt -> 200
Content-Type -> text/plain
В ответе есть: Sitemap: https://by-population-maps.vercel.app/sitemap.xml
```

### 2. Добавить XML sitemap

**Проблема.** URL <https://by-population-maps.vercel.app/sitemap.xml> возвращает `404` и HTML-страницу ошибки.

**Риск.** Роботам сложнее быстро найти исследования, страницы артефактов и белорусские версии; новые материалы могут попадать в индекс заметно медленнее.

**Просьба к команде.** Создать `web/app/sitemap.ts` или эквивалентный генератор. Включать только индексируемые канонические страницы: главные, исследования, статьи, методологию, автора и публичные страницы артефактов на русском и белорусском языках.

**Критерии приёмки.**

```text
GET /sitemap.xml -> 200
Content-Type -> application/xml или text/xml
В документе есть абсолютные URL и отсутствуют 404/служебные URL
```

## P1 — влияет на качество индексации и выбор страницы в выдаче

### 3. Указать canonical URL для всех индексируемых страниц

**Проблема.** На проверенных страницах, включая `/`, `/about`, `/be/about`, `/map`, `/research/aging` и `/artifacts/aging`, нет тега `<link rel="canonical">`.

**Риск.** Поисковая система сама выбирает основной URL при похожем содержимом, параметрах ссылок и двуязычных версиях.

**Просьба к команде.** Добавить `alternates.canonical` в metadata каждой страницы. Логику построения абсолютных URL лучше централизовать в `web/lib/seo.ts`.

**Критерии приёмки.** Каждая индексируемая страница содержит один абсолютный canonical URL на саму себя.

### 4. Завершить разметку `hreflang`

**Проблема.** `hreflang` присутствует, например, на `/be/about` и `/be/research/aging`, но отсутствует на главных страницах и картах (`/`, `/be`, `/map`, `/be/map`) и на русских страницах исследований, например `/research/aging`.

**Риск.** Google и Яндекс могут показывать пользователю не ту языковую версию либо воспринимать переводы как недостаточно связанные.

**Просьба к команде.** Для каждой пары переводов добавить обе взаимные ссылки `hreflang="ru"` и `hreflang="be"`, а также `hreflang="x-default"` на выбранную основную версию. Все URL должны быть абсолютными и возвращать `200`.

**Критерии приёмки.** На каждой переведённой странице есть полный, взаимный набор `ru`, `be`, `x-default`.

### 5. Исправить язык HTML-документа для белорусской версии

**Проблема.** Корневой layout в `web/app/layout.tsx` всегда выводит `<html lang="ru">`; белорусские страницы содержат `lang="be"` только на вложенном контейнере.

**Риск.** Это ухудшает сигнал о языке документа для поисковых систем, браузеров и ассистивных технологий.

**Просьба к команде.** Перестроить layout так, чтобы URL под `/be/` отдавались с `<html lang="be">`, а русские — с `<html lang="ru">`.

**Критерии приёмки.**

```text
GET /about     -> <html lang="ru">
GET /be/about  -> <html lang="be">
```

### 6. Оптимизировать SEO-шаблон страницы карты

**Проблема.** У `/map` нет собственного metadata и H1: она наследует title и description главной. В начальном HTML карты нет содержательного заголовка — преимущественно интерфейс и состояние загрузки данных.

**Риск.** Страница карты не получает самостоятельную релевантность по запросам вроде «карта населения Беларуси», «численность населения районов Беларуси» и «плотность населения Беларуси».

**Просьба к команде.** В `web/app/map/page.tsx` добавить уникальные title, description, H1 и компактный статичный текст с покрытием карты: годы, административные уровни, показатели и ссылки на методологию/данные. Аналогично проверить `/be/map`.

**Критерии приёмки.** Карта имеет уникальные title, meta description, H1, canonical и `hreflang`; её основное назначение доступно в HTML без выполнения JavaScript.

## P2 — улучшает расширенные сниппеты и тематическую релевантность

### 7. Добавить schema.org JSON-LD

**Проблема.** JSON-LD есть на странице автора, но отсутствует на главной, исследованиях и страницах наборов данных.

**Просьба к команде.** Добавить:

- `WebSite` и `Organization` — на главной;
- `Article` или `ScholarlyArticle` — на страницах исследований и большой статье;
- `Dataset` — на страницах открытых наборов/артефактов;
- ссылки `author`, `publisher`, `datePublished`/`dateModified`, `inLanguage`, `sameAs` там, где данные достоверны.

**Критерии приёмки.** Разметка проходит проверку в Schema Markup Validator без критических ошибок.

### 8. Сделать title страниц артефактов более поисковыми

**Проблема.** Названия вида «Пакет aging — версии и состав» используют внутренний технический slug вместо предметной темы.

**Просьба к команде.** Использовать предметные формулировки, например: «Старение районов Беларуси: данные, методика и воспроизводимый пакет».

**Критерии приёмки.** Каждый title отражает тему исследования, географию и тип материала; внутренние коды остаются вторичным элементом.

## После релиза

1. Добавить сайт в Google Search Console и Яндекс Вебмастер.
2. Отправить `https://by-population-maps.vercel.app/sitemap.xml` в обе системы.
3. Запросить переобход главной, `/map`, одного исследования и их белорусских версий.
4. Ежемесячно проверять отчёты об индексировании, исключённых URL, canonical и языковых страницах.

## Команды проверки после выкладки

```bash
curl -i https://by-population-maps.vercel.app/robots.txt
curl -i https://by-population-maps.vercel.app/sitemap.xml
curl -s https://by-population-maps.vercel.app/map | rg 'canonical|hrefLang|<h1'
curl -s https://by-population-maps.vercel.app/be/about | rg '<html lang="be"|canonical|hrefLang'
```

## Ограничения аудита

Проверка выполнена по доступной публичной версии сайта и исходному коду. Она не включает данные Google Search Console, Яндекс Вебмастера, журналов обхода, фактические позиции или показатели Core Web Vitals из полевых данных.
