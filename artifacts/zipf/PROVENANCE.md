# Происхождение данных: цепочка источник → преобразование → результат

```
pop-stat.mashke.org/belarus-cities.htm          (обращение 2026-07-11)
  │  компиляция Tim Bespyatov по официальным публикациям:
  │  переписи 1897/1923/1926/1939/1959/1970/1979/1989/1999/2009/2019,
  │  текущие оценки Белстата 1991, 2004-2026
  ▼
sources/raw/ps_cities.html                      (sha256 в sources/registry.csv)
  │  code/build.py::parse_cities
  │  - разбор HTML-таблицы (незакрытые <td>, значения в тысячах)
  │  - перепись имеет приоритет над оценкой того же года
  │  - строки-заголовки областей и примечания отбрасываются
  ▼
внутренние ряды {город: {год: население}}       (222 города)
  │  code/build.py::main
  │  - для каждого среза YEARS: сортировка по убыванию населения
  │  - ранжировка (без интерполяции: только годы, присутствующие в источнике)
  ▼
data/final/zipf_ranks.csv                       (1945 строк, 12 срезов)
  │  code/build.py::gi_fit
  │  - регрессия log(rank - 1/2) ~ log(pop) по топ-N (Габэ-Ибрагимов 2011)
  │  - N = 20/30/50; SE = |b|*sqrt(2/N)
  ▼
data/final/zipf_slopes.csv, primacy.csv, computed_results.json
  │  etl/zipf.py (тот же алгоритм в репозитории проекта; эквивалентность
  │  двух реализаций закреплена pytest-тестом в CI)
  ▼
web/public/data/zipf.json -> https://by-population-maps.vercel.app/research/zipf
```

Никакие значения не вводились вручную; единственный ручной шаг —
фиксация допущений в `params/assumptions.yaml`.
