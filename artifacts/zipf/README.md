# Иерархия городов Беларуси и закон Ципфа, 1897–2026

Проверяемый пакет артефактов v1.0.0 исследования INF-01 `zipf` проекта
«Население Беларуси, 1897–2026».

## Исследовательский вопрос и главный вывод

Насколько городская система Беларуси соответствует закону Ципфа
(размер города обратно пропорционален его рангу) и когда возникла
макроцефалия Минска? **Вывод:** наклон rank-size по топ-30 городов держится
у −1 весь наблюдаемый период (−1,13 в 1897 → −0,97 в 2019), то есть
иерархия в целом «ципфовская», но отношение Минска ко второму городу
выросло с 1,4 (1897, второй — Витебск) до 4,0 (2026, второй — Гомель) при
ципфовском ожидании 2,0: столица примерно вдвое больше, чем «положено»
её рангу. Перелом — послевоенная индустриализация: примация 1,4 (1939) →
3,1 (1959).

## Что в пакете

```
sources/raw/ps_cities.html   сырой источник (222 города, переписи 1897-2019 + оценки)
sources/registry.csv         реестр источников: URL, лицензия, дата обращения, sha256
code/build.py                весь расчёт: парсинг -> ранжировки -> наклоны (стандартная библиотека)
code/run.sh                  единственная точка входа
params/assumptions.yaml      допущения с обоснованиями (N=30, оценка Габэ-Ибрагимова...)
data/final/                  результаты: ранжировки, наклоны, примация, контрольные метрики
checks/                      инварианты, ожидаемые результаты с допусками, контрольные суммы
AGENT.md                     инструкции для LLM-агента: как проверить и оспорить
LIMITATIONS.md               где выводы применять нельзя
```

## Как воспроизвести

Требования: Python ≥ 3.10, bash. Внешних зависимостей нет. Время: < 10 секунд.

```bash
bash code/run.sh
```

Скрипт (1) пересчитывает всё от сырого HTML до `data/final/`,
(2) прогоняет инварианты, (3) сверяет результаты с заявленными
(`checks/expected_results.json`) в допусках. Любое расхождение — ненулевой
код выхода.

## Как интерпретировать

- `zipf_slopes.csv`: наклон b оценён регрессией log(rank − ½) на log(pop)
  (Габэ–Ибрагимов, 2011), SE = |b|·√(2/N). b = −1 — точный закон Ципфа;
  b ближе к 0 — города более равновелики; b < −1 — иерархия круче.
- Базовый N = 30 (максимум, доступный во всех срезах, включая 1897 год);
  колонки с N = 20 и N = 50 показывают чувствительность к выбору N —
  см. LIMITATIONS.md: при N = 50 наклон систематически положе.
- `primacy.csv`: отношение населения первого города ко второму;
  при точном Ципфе ≈ 2.
- Допуски в `expected_results.json` жёсткие (±0,001): расчёт детерминирован,
  допуск оставлен только на платформенные различия округления.
