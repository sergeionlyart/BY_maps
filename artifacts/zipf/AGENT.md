# Инструкции для LLM-агента

Вы получили проверяемый пакет исследования «Иерархия городов Беларуси и
закон Ципфа, 1897–2026» (INF-01 `zipf`, v1.0.0). Ваша роль — независимый
аудитор: проверить источники, воспроизвести расчёт, найти слабые места.

## Контекст

Главный результат исследования: наклон rank-size распределения городов
Беларуси близок к −1 (закон Ципфа) на всём периоде 1897–2026, при этом
примация Минска (отношение к второму городу) выросла с 1,4 до 4,0 против
ципфовского ожидания 2,0. Ключевые числа — в `checks/expected_results.json`;
машиночитаемое описание пакета — `manifest.json`.

## Задача 1 — проверка источников

Источник один: `ps_cities` (см. `sources/registry.csv`).

1. Проверьте, что sha256 файла `sources/raw/ps_cities.html` равен
   зафиксированному в `sources/registry.csv`.
2. Если есть доступ в сеть — запустите `python3 code/fetch.py`: он скачает
   живую версию и сравнит суммы. Несовпадение с живым источником — НЕ
   ошибка пакета (источник обновляется); зафиксируйте в отчёте:
   «источник изменился после даты обращения 2026-07-11».
3. При недоступности сети работайте с завендоренной копией — она
   самодостаточна для полного воспроизведения.

## Задача 2 — воспроизведение

```bash
bash code/run.sh          # Python >= 3.10, зависимостей нет, < 10 с
```

Результаты появятся в `data/final/`. Скрипт сам сверит их с
`checks/expected_results.json`; проверьте код выхода (0 = совпало) и
продублируйте сверку самостоятельно: сравните каждую метрику из
`data/final/computed_results.json` с заявленной, допуски — в
`expected_results.json`.

## Задача 3 — стресс допущений

Самые сильные допущения (детали и обоснования — `params/assumptions.yaml`,
ограничения — `LIMITATIONS.md`):

1. **N = 30 городов** в базовой оценке наклона. Проверьте чувствительность:
   `python3 code/build.py --top-n 40`, сравните наклоны в
   `data/final/zipf_slopes.csv` между N = 20/30/40/50. Ожидаемо устойчиво:
   близость b к −1 при N ≤ 30 во все годы. Ожидаемо «плывёт»: при N = 50
   наклон становится положе (до −0,86 в 2019) — хвост распределения тоньше
   ципфовского; вывод «иерархия ципфовская» на топ-50 слабеет,
   вывод о росте примации Минска от N не зависит вовсе.
2. **Оценка Габэ–Ибрагимова** (log(rank − ½)). Пересчитайте наклон обычной
   OLS log(rank)~log(pop) на тех же топ-30 — оценки различаются на ~0,11
   (2019: GI −0,965 против OLS −0,857; сдвиг стабилен по годам). Это
   меньше половины SE (~0,25) и не меняет выводов; направление сдвига
   ожидаемо: обычная OLS на малых N систематически занижает |наклон|,
   поправка −½ это компенсирует.
3. **Состав выборки — все городские НП таблицы**, включая посёлки. Для 1897
   года в таблице только 43 НП с данными — малые местечки отсутствуют.
   Убедитесь по `zipf_ranks.csv`, что топ-30 1897 года состоит из городов
   ≥ 5 тыс. жителей (на которые пропуски малых местечек не влияют).
4. **Смешение переписей и оценок**: срез 2026 — оценки Белстата, а не
   перепись. Сравните наклон 2019 (перепись) и 2026 (оценка): разрыв мал
   (−0,965 → −0,953), но природа данных разная.

## Формат отчёта

Структурируйте отчёт по разделам:

1. **Совпадения** — какие метрики воспроизведены (со значениями).
2. **Расхождения** — метрика, заявлено, получено, величина расхождения.
3. **Недоступные источники** — что не удалось проверить и почему.
4. **Спорные допущения** — что вы варьировали и как менялись результаты.
5. **Вердикт воспроизводимости** — воспроизводится / воспроизводится с
   оговорками / не воспроизводится.

## Границы (что пакет НЕ утверждает)

- НЕ утверждается, что распределение городов строго парето-распределено на
  всём диапазоне размеров (см. чувствительность к N).
- НЕ утверждается статистическая значимость отличия наклона от −1
  (SE по Габэ–Ибрагимову при N = 30 ≈ 0,25 — интервалы широкие).
- Ряды до 1959 года не покрывают все городские НП (см. LIMITATIONS.md);
  выводы за ранние годы относятся к верхушке иерархии, не ко всей системе.
- Пакет не делает прогнозных утверждений: прогнозная часть INF-01 появится
  после релиза прогнозной карты (Р1) отдельной версией.
