# ML-challenger: где структурная модель районов систематически ошибается (проверяемый пакет)

Градиентный бустинг (чистый Python, без sklearn/scipy/numpy) как
**challenger** структурной модели районов CCR/Гамильтона-Перри — по
мандату ROADMAP §2: «только для проверки, где статистические модели
систематически ошибаются; 50-летний горизонт — вне области применения
ML». Это **диагностика на 7-летнем окне, а не конкурирующий прогноз** и не
инструмент долгосрочного прогнозирования; инкумбент CCR/CCMPP остаётся
основой.

## Главный вывод

Инкумбент CCR **систематически ошибается предсказуемым образом**: на его
единственном подлинно out-of-sample окне (база-2019 → факт-2026) бустинг
предсказывает **33% дисперсии** знаковой ошибки CCR вне выборки (OOF
R²=0,33; повторная CV 0,30–0,34; перестановочный нуль p=0,005 — далеко за
нуль-полосой). Из них ~20 п.п. объясняет структура, которую CCR и так
использует, и **ещё +13 п.п. — экзогенные сигналы, которые CCR
структурно не видит**. Доминирует **миграционное сальдо 2015–2019**
(единственный признак с важностью явно вне нуль-полосы); зарплатный и
транспортный градиенты добавляют слабо. Содержательно: **модель когортных
коэффициентов не видит миграционную динамику** — она недооценивает
пригородные районы с высоким притоком (Смолевичский +8,7%) и переоценивает
районы, где она экстраполировала прошлый бум (Минский −9,2%). Коррекция ML
снижает MAPE распределения CCR с 2,19% до 1,67% (наивная 2,84%), но это
**понижено** под диагностику — веер прогноза сюда не подмешан.

**Честная оговорка**: мишень — ошибка CCR относительно ОФИЦИАЛЬНОЙ оценки
2026, а не «истины». Официальная оценка сама учитывает зарегистрированную
миграцию 2019–2025, поэтому обнаруженная систематическая ошибка означает
прежде всего «CCR недо-использует миграционные данные». Одно только
миграционное сальдо даёт OOF R²≈0,31 ≈ всю модель (0,33) — сигнал почти
целиком миграционный, а перепись→перепись окно 2009→2019 (без миграционных
признаков, R²≈0,02) НЕ проверяет его против настоящего счёта. Магнитуда —
слабое свидетельство (окно 7 лет, мишень — сглаженная оценка со СКО лог-
прироста лишь 0,055; выигрыш MAPE 0,5 п.п. на n=118 в пределах CV-шума);
надёжна именно диагностика «где и по какому сигналу», не число.

## Что именно сделано

**Мишень** — знаковая ошибка CCR: `e_i = ln(факт_2026_i / CCR_2026_i)`,
взятая ровно из `etl/forecast/backtest_sub.py` (Гамильтон-Перри обучен на
2009→2019, спрогнозирован на 2026 БЕЗ калибровки старта, нормирован к
фактическим областным итогам). Поскольку CCR нормирован к истинному
областному итогу, `e_i` — **чисто внутриобластная** ошибка распределения.

**Признаки** (12, все датированы ≤ 2019, интенсивные/периметр-устойчивые):
экзогенные, которые CCR игнорирует — зарплата к среднереспубликанской
(2019) и её рост 2015→2019, транспортная доступность и доступ к границе ЕС
(2019), душевая ночная светимость и её тренд 2015→2019, миграционное
сальдо 2015–2019; структурные контроли — доли 0–14 и 65+, ln населения,
чернобыльский класс, host-флаг. Ни один признак не из 2020–2026 (защита от
утечки; исключены все оконно-посчитанные поля INF-джейсонов).

**Модель** — градиентный бустинг регрессионных пней (глубина 2, lr 0,05,
листья ≥10, subsample 0,7, подвыборка 5 из 12 признаков), число деревьев
выбрано ранней остановкой по кросс-валидации (42). Сеяно (SEED=20260712) →
байт-воспроизводимо.

**Честность встроена**: LOO-по-областям CV (6 фолдов; Минск-город — не
район); **перестановочный нуль** (200 тасовок мишени при фиксированном m)
гейтит каждое заявление; повторная CV (30 сидов) даёт полосу; гонка MAPE
понижена; вклады аддитивно раскладываются по признакам (карта «слепых
пятен»). Вторичное окно 2009→2019 (перепись-золото, только структурные
признаки) даёт OOF R²≈0,02 — подтверждает, что **сама структура едва
предсказывает**, а содержание несёт экзогенный сигнал (доступный лишь на
базе-2019).

## Как воспроизвести

Требования: Python ≥ 3.10, только стандартная библиотека. Время ~5 минут.

```bash
bash code/run.sh
```

Скрипт: (1) собирает панель, импортирует CCR-прогноз из движка `forecast`,
считает бустинг + LOO-CV + перестановочный нуль (200) + повторную CV;
(2) инварианты (в т.ч. на перемешанной мишени OOF R² падает в нуль-полосу
— пайплайн не выдумывает сигнал); (3) сверку с `checks/expected_results.json`
(16 метрик). Прогон детерминирован: повтор даёт байт-в-байт те же числа.
