# Инструкция для ИИ-агента: аудит пакета INF-02 «Старение районов»

Ты работаешь с проверяемым пакетом исследования. Всё, что заявлено на
лендинге, обязано воспроизводиться отсюда одной командой.

## Быстрый старт

```bash
bash code/run.sh   # Python >= 3.10, стандартная библиотека, ~10 с
```

Ожидаемый финал: `Все 10 контрольных метрик воспроизведены в допусках.`
Код выхода 0 = пакет честен; не 0 — расхождение, о котором стоит сообщить.

## Что проверять

1. **Детерминизм**: два запуска подряд дают байт-в-байт одинаковый
   `web/public/data/aging.json` (случайности в коде нет).
2. **Сходимость с переписями**: суммы пирамид по 7 регионам равны
   официальным итогам (допуск ≤17 чел. — поэлементное округление 34 ячеек;
   записи «возраст не определён» переписи-2009 — 296 человек по стране —
   распределены пропорционально — см. `etl/aging.py:load_structures`).
3. **Инварианты** — `checks/tests/test_invariants.py` (запускается в run.sh).

## Стресс-тесты допущений (params/assumptions.yaml)

Перед каждым перезапуском с правкой кода удаляй байткод-кэш
(`rm -rf etl/__pycache__ etl/forecast/__pycache__`), иначе правка,
попавшая в ту же секунду mtime, может молча не примениться.

- **Порог 30%**: замени `THRESHOLD_65` в `etl/aging.py` на 0.25/0.35 и
  перезапусти. Ожидание: число пересекающих районов резко меняется
  (0,25 → 100 из 118; 0,30 → 28; 0,35 → 2; множества строго вложены),
  ранжирование по «годам до порога» сохраняется приближённо (ранговая
  корреляция между порогами ≈ 0,8), связь с долей 65+ 2019 отрицательная
  (ранговая ≈ −0,7).
- **Смертность контрфакта**: `E0_BASE` ±1 год — состав пограничных районов
  и число пересекающих порог заметно меняются (28 → от 19 при e0−1 до 34
  при e0+1; отдельные районы сдвигаются на несколько шагов), но знак
  главного вывода устойчив: естественная убыль в 117 районах из 118 при
  всех вариациях.
- **Рождаемость**: контрфакт использует областной ASFR-профиль 2018 г.
  при национальном СКР базового сценария; подстановка СКР 1,2/1,7 меняет
  темпы, но не знак естественной динамики в периферийных районах.

## Ловушки

- `etl/forecast/` здесь — подмножество пакета by-maps-forecast: aging
  использует только `data.py:mortality_mx/asfr_profile`, `lifetable`,
  `ccmpp`. Функции `jumpoff_2026`, `migration_matrix`, `wpp_*` ссылаются
  на входы, которых в этом пакете нет, и упадут при вызове — это не
  дефект воспроизводимости, run.sh их не использует.
- Открытая группа «80+» в медиане условно ограничена 100 годами — влияет
  только на территории с медианой >77,5 (таких нет).
- Контрфакт — диагностика структуры, НЕ прогноз: сравнение с реальными
  траекториями после 2019 г. некорректно без миграции.
- ASFR района = ASFR его области (районных ASFR не существует в открытом
  доступе) — уровень района смещён, знак и порядок — нет.
