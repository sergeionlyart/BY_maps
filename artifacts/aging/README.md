# Старение районов Беларуси: проверяемый пакет (INF-02, v1.0.0)

Пакет воспроизводит все числа исследования
«[Старение районов](https://by-population-maps.vercel.app/research/aging)»:
медианный возраст, долю 65+, коэффициент демографической нагрузки по
переписям 2009/2019 для 118 районов, 12 городов областного подчинения и
7 регионов — и контрфактную когортную передвижку «при нулевой миграции»
(лет до порога 30% доли 65+, естественная динамика 2019–2039).

## Воспроизведение одной командой

```bash
bash code/run.sh
```

Требования: Python ≥ 3.10, только стандартная библиотека (~10 с).
Скрипт пересчитывает индикаторы из переписных возрастных структур
(`data/curated/age2009.csv`, `age2019.csv`), прогоняет контрфактную
передвижку движком CCMPP (тот же, что в пакете прогноза by-maps-forecast),
проверяет инварианты и сверяет результат с заявленными метриками
(`checks/expected_results.json`).

## Главные результаты

- **117 из 118 районов** ждёт естественная убыль даже при полном
  прекращении миграции: возрастная структура делает депопуляцию
  самоподдерживающейся.
- **28 районов** пересекут порог 30% доли 65+ в горизонте 60 лет,
  из них 20 — в ближайшие 20 лет.
- Самый «старый» район — **Свислочский** (26,0% жителей 65+, медианный
  возраст 51,9 года; порог 30% — примерно в 2029 г.).
- Даже Минск (медиана 37,8) в контрфакте без миграции теряет
  0,56%/год: рост столицы держится только на притоке.

## Структура пакета

- `data/curated/` — входные данные (переписные структуры, смертность,
  областные ASFR) и итоговый `aging_indicators.csv`;
- `etl/` — код: `aging.py` + движок передвижки (`forecast/`);
- `params/assumptions.yaml` — все допущения с обоснованиями;
- `web/public/data/aging.json` — файл, который читает лендинг;
- `checks/` — контрольные метрики и инварианты; `code/verify.py` — сверка.

Методика, ограничения и инструкции для агентов — `LIMITATIONS.md`,
`PROVENANCE.md`, `AGENT.md`.
