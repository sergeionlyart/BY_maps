# Цена пустеющей карты: материальный навес городов Беларуси, 1975–2020

Проверяемый пакет артефактов v1.0.0 исследования INF-12 `urban-overhang`
проекта «Население Беларуси, 1897–2026».

## Исследовательский вопрос и главный вывод

Что происходит с физическим размером, материальным фондом и инфраструктурной
географией городов Беларуси, когда их население сокращается? **Вывод:** из
92 городов панели качества A/B 1990–2020 (реестр — 94 города с населением
≥10 000 хотя бы в один опорный год; Фаниполь и Мачулищи — класс C и в панель
не входят, у Мачулищей нет точки 1990) население сокращалось в 39;
у **всех 39** площадь накопленной застройки на жителя (фонд на жителя, BPC)
устойчиво росла — темп материального навеса `MOR = BGR − PGR` положителен во
всех 9 сценариях границ (пороги 5/10/20 % × замыкание 0/100/200 м) и выше
минимально обнаружимого изменения. Медианный `MOR` сокращающихся городов
**+1,83 %/год** (1990→2020); медианный фонд на жителя панели за тот же интервал
вырос с **67,3 до 119,4 м²/жителя**. Это и есть «материальный навес» — больше
физического города на меньшее число постоянных жителей.

Вывод сопровождается тремя обязательными оговорками, без которых его нельзя
цитировать:

1. **GHSL монотонен.** Исторический GHS-BUILT-S — накопительный ряд: продукт
   не измеряет снос и физическое сжатие. «Отрицательный навес» в нём в принципе
   не может возникнуть иначе как за счёт роста населения, а тип «физическое
   сокращение города» не присваивается по базовому продукту. Часть роста фонда
   между эпохами может быть сенсорной (разные поколения Landsat/Sentinel);
   консервативная ошибка по классу размера это учитывает, но не устраняет.
2. **Навес — это не «особое строительство» в сокращающихся городах.** `MOR>0`
   во всех 93 городах, для которых интервал измерим (92 панели A/B плюс
   Фаниполь класса C), включая растущие (минимальный `MOR` +0,22 %/год
   у Заславля — и он единственный не проходит порог устойчивости).
   У сокращающихся навес создаётся сочетанием общенационального роста
   накопленного фонда со сжатием знаменателя (числа жителей); разрыв `MOR`
   с сопоставленными «двойниками» со стабильным населением — **+0,24 п.п./год**
   (медиана по 31 паре) — статистически неотличим от нуля (знаковый тест 21/31,
   p = 0,071) и публикуется только как описательный. Числитель и знаменатель
   во всех выводах и визуализациях показываются раздельно.
3. **«Цена» — метафора.** Пакет измеряет удельную материальную и
   инфраструктурную нагрузку, а не денежную стоимость содержания города.
   Денежные утверждения («обходится бюджету на X % дороже») запрещены
   редакционным контрактом и в пакете отсутствуют.

Типология траекторий (правиловая, интервал 1990–2020, панель 92 городов):
T1 «компактный рост» — 38, T3 «стабильная оболочка при депопуляции» — 1,
T4 «расширение во время депопуляции» — 38, TX «неопределённая траектория»
(стабильное население либо знак MOR не проходит гейт устойчивости) — 15.

    morph_qa.csv               санитарные контроли (вода, лес, болото)

    monotowns.csv              перечень моногородов (контекст INF-06)

  city_metrics.csv             город × год × рамка: BPC, плотность, свет/фонд…

Полные глобальные растры GHSL (~0,8 ГБ) и снимок OSM (~0,35 ГБ) в пакет **не
вкладываются** — в реестрах остаются URL + sha256, а `code/fetch.py`
воспроизводимо их скачивает и сверяет. Быстрый режим стартует от вендоренных
зональных агрегатов и сети не требует.

## Как воспроизвести

Требования быстрого режима: Python ≥ 3.10, bash. Внешних зависимостей нет
(только стандартная библиотека). Время: секунды.

```bash
bash code/run.sh
```

Скрипт (1) пересчитывает `data/final/` из `sources/raw/` (`code/build.py`),
(2) прогоняет инварианты (`checks/tests/test_invariants.py`), (3) сверяет
результаты с заявленными (`checks/expected_results.json`) в допусках
(`code/verify.py`). Любое расхождение — ненулевой код выхода.

### Режимы quick и full

- **quick** (по умолчанию, `bash code/run.sh`) — детерминированный пересчёт
  национальных показателей от завендоренных зональных агрегатов до `data/final/`
  на стандартной библиотеке. Проверяет всю арифметику метрик, типологию,
  сопоставление пар и контрольные числа, но принимает агрегаты фонда, света,
  дорог и POI как вход.

- **full** — воссоздание самих агрегатов из первичных данных. Требует
  `rasterio`, `numpy`, `scipy`, `pyosmium`, `shapely`, `pillow` (версии — в
  `code/requirements.lock`) и запускается из корня репозитория BY Maps. Шаги:

  ```bash
  python3 code/fetch.py --ghsl --osm      # 40 тайлов GHSL + Geofabrik PBF (sha256)
  python -m etl.urban_extract             # мозаика+клип GHSL, фонд города×эпоха
  python -m etl.urban_morph               # морфологические контуры, 3 рамки, ядро/край
  python -m etl.urban_light               # зональный свет (вырезки INF-08), 2012–2024
  python -m etl.urban_osm all             # дороги, POI, здания, админ-границы из PBF
  ```

  На выходе — те же `sources/raw/*.csv`, после чего `bash code/run.sh`
  повторяет быстрый режим. Внимание: `belarus-latest.osm.pbf` у Geofabrik —
  живой файл; несовпадение sha256 при новой загрузке не ошибка пакета, а сигнал,
  что снимок обновился после даты обращения (2026-07-16) и нужен новый релиз.

## Живая версия и аудит

- Лонгрид и интерактивы: <https://by-population-maps.vercel.app/research/urban-overhang>
- Методблок: там же, кнопка «О данных и методике»

- Репозиторий: <https://github.com/sergeionlyart/BY_maps>

Пакет рассчитан на независимую проверку человеком или LLM-агентом.
Проверяемые задания — воспроизведение, ручной пересчёт `MOR` для пяти городов,
повтор классификации при двух альтернативных границах, поиск неподдержанных
денежных/причинных утверждений, проверка контрпримера и лицензий — в `AGENT.md`.
Границы применимости и альтернативные объяснения — в `LIMITATIONS.md`.
