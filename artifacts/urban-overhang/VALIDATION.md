## 2. Монотонность накопленного фонда

Продукт заявлен как накопительный; проверено фактически: в
`sources/raw/morph_flows.csv` (94 города × 9 межэпоховых интервалов ×

      sum(1 for r in rows if float(r['negative_m2'])>0))"

## 7. Минимально обнаружимое изменение (MDC) по классам размера

MDC для MOR: `sqrt(2) · rel_err / 30` (две независимые ошибки уровня фонда на
концах 30-летнего интервала); rel_err задан консервативно по классу размера
(`params/assumptions.yaml`, значения шире глобальных оценок точности продукта):

| Класс (фонд 2020) | rel_err | MDC, п.п./год | Городов в панели |
|---|---:|---:|---:|
| ≥ 10 км² | 5 % | 0,24 | 10 |
| 1–10 км² | 10 % | 0,47 | 77 |
| < 1 км² | 20 % | 0,94 | 5 |

`robust_sign = 1` (знак MOR одинаков во всех 9 сценариях И |MOR| > MDC) —
у **92 из 93** городов с метриками главного интервала (91 из 92 панели A/B
плюс Фаниполь класса C). Единственное исключение — Заславль (растущий
город: MOR +0,22 %/год < MDC 0,47 п.п./год, и 2 из 9 сценариев дают слабо
отрицательное значение). Все 39 сокращающихся городов превышают свой MDC;
медианный MOR сокращающихся (+1,83 %/год) почти вчетверо выше типичного MDC
(0,47 п.п./год) и вдвое выше самого широкого (0,94 — класс < 1 км², 5 из 39
сокращающихся). Оговорка о смысле порога: MDC — floor обнаружимости порядка
~1σ сценарно-измерительного шума, множитель уровня доверия не применялся
(METHODS §9.2).

```bash
python3 -c "
import csv
rows=[r for r in csv.DictReader(open('data/final/city_interval_metrics.csv'))
      if r['year_start']=='1990' and r['year_end']=='2020']
from collections import Counter
print(Counter(r['minimum_detectable_change'] for r in rows))
print('robust_sign=1:', sum(1 for r in rows if r['robust_sign']=='1'), 'из', len(rows))"
```

## 10. Фонд GHSL против зданий OSM

Замена невыполненного ручного аудита (§13): корреляция Пирсона между
застроенной поверхностью GHSL 2020 года (рамка `t10_c1`, MORPH_FIXED_FRAME)
и числом зданий OSM по 94 городам — **r = 0,953** (в лог-лог координатах —
**0,961**). Два независимых по происхождению слоя (спутниковая классификация
против ручного картирования) дают согласованный рельеф размеров городов.

```bash
python3 -c "
import csv, math
bld={r['city_id']:float(r['buildings_count'])
     for r in csv.DictReader(open('sources/raw/city_buildings.csv'))
     if r['city_id']!='__national__'}
b={r['city_id']:float(r['built_fixed_m2'])
   for r in csv.DictReader(open('sources/raw/morph_fixed.csv'))
   if r['scenario']=='t10_c1' and r['epoch']=='2020'}
ids=[c for c in bld if c in b]
xs=[math.log(b[c]) for c in ids]; ys=[math.log(bld[c]) for c in ids]
n=len(ids); mx=sum(xs)/n; my=sum(ys)/n
r=(sum((x-mx)*(y-my) for x,y in zip(xs,ys))
   /(sum((x-mx)**2 for x in xs)**.5*sum((y-my)**2 for y in ys)**.5))
print('n =', n, ' r(log-log) =', round(r,3))"
```

## 12. Внутренний adversarial-аудит: дефекты, пойманные и исправленные до публикации

Перед релизом пакет прошёл внутренний adversarial-аудит; три найденных
дефекта исправлены, все данные пересчитаны, поправки зафиксированы
датированными записями A1–A3 в PREREGISTRATION.md (плюс поправка о
контрпримере). Примеры пойманного:

1. **Коллапс привязки Бреста (поправка A2).** Первичное правило привязки
   города к компоненту («ближайшая размеченная ячейка в радиусе 2 км»)
   перепривязывало город к мелкому валидному осколку: контур Бреста
   «схлопывался» с 3 688 ячеек (1980) до 15 (1985) — абсурд для монотонного
   продукта. Правило заменено на «крупнейший валидный компонент в окне
   2 км»; после исправления контур Бреста монотонен: 36,3 км² (1975) →
   67,0 км² (2020).
2. **Рассинхрон зон «края» (поправка о зонах).** Буферные ячейки фикс-рамки
   входили в «край» фонда, но не в «край» света. Зоны разделены строго:
   ядро / край / буфер (`built_buffer_m2` в `morph_fixed.csv`), маски фонда
   и света согласованы (§4).
3. **Геопривязка света.** Перенос световых растров на суперсетку опирался
   на предположение о выравнивании сеток и давал сдвиг до 4 суперячеек;
   заменён на reproject по фактическим трансформациям растров.

Также по итогам аудита ужесточены формулировки: гейт типологии §10.3
(Заславль → TX), национальные агрегаты только по панели A/B (A3), статусы
гипотез H-1/H-2 приведены к замороженным критериям (METHODS §1).

## 13. Что НЕ провалидировано (честный перечень)

1. **Ручной аудит 24 городов по высокодетальным снимкам (ТЗ §12.1) в MVP не
   выполнялся**: стратифицированная выборка, двое независимых разметчиков и
   третий рецензент требуют ресурсов за пределами MVP. Замена: консервативные
   rel_err 5/10/20 % (заведомо шире глобальных оценок точности GHSL) в составе
   MDC (§7) + корреляция с OSM-зданиями (§10). Следствие: локальные ошибки
   продукта в отдельных городах (особенно в эпохах 1975–1990, где OSM-сверка
   невозможна) не исключены; знак главного вывода защищён запасом
   медианы над MDC, точные значения MOR отдельных городов — нет.
2. **Полнота POI не сверена с официальными реестрами** (recall не измерен) —
   гейт ТЗ §12.4 не пройден, поэтому изохроны доступности не рассчитываются
   и не публикуются; POI показываются только как счётчики с бейджем полноты.
3. **RES/NRES-декомпозиция** фонда недоступна (план v1.1); промышленная
   застройка внутри контуров не отделена от жилой.
4. **Monte-Carlo-перебор неопределённости (ТЗ §12.6)** заменён
   детерминированным перебором 9 сценариев границ + admin-рамка + MDC;
   стохастические компоненты (интерполяция населения, ошибки эпох) отдельно
   не сэмплировались.
5. **Эпохи 1975–1990 GHSL** не имеют независимой внешней сверки в пакете
   (OSM и VNL покрывают только современность; DMSP начинается с 1992 года).
