# Прогноз населения Беларуси 2026–2075, v2026.1 (проверяемый пакет)

Этап MVP: страна, 6 областей, Минск; три детерминированных сценария;
80% интервал для базового.

## Главный вывод

При любом из трёх сценариев население Беларуси продолжит сокращаться:
с 9,06 млн (01.01.2026) до 7,53 млн к 2050 в базовом сценарии
(80% интервал 6,99–8,09) и до 5,97 млн к 2075 (5,04–6,99). Оптимистический
сценарий даёт 7,11 млн к 2075, негативный (продолжение эмиграционной
волны) — 4,33 млн. Доля Минска в населении страны растёт при любом
сценарии: столица сокращается медленнее регионов (сильнейшая убыль —
Витебская область).

## Методика (кратко)

Когортно-компонентный метод (CCMPP), 5-летние группы, шаг 5 лет, раздельно
по областям (страна = сумма). Рождаемость: областные ASFR-профили 2018 г.
(ежегодник Белстата), масштабируемые к сценарной национальной траектории
СКР. Смертность: таблицы дожития HMD-2018, масштабируемые к сценарным e0.
Миграция: внутренняя — профиль межобластной матрицы переписи-2019 с
калиброванным объёмом; международная — сценарное сальдо по ключам
концентрации. Базовый сценарий следует допущениям UN WPP 2024 medium
(траектории в `data/raw/wpp2024/blr_indicators_medium.csv`); квантили —
относительная ширина 80% PI WPP. Все допущения — `params/assumptions.yaml`
и `etl/forecast/scenarios/*.yaml` (с обоснованием каждого значения).

## Как воспроизвести

Требования: Python ≥ 3.10, PyYAML (`pip install -r code/requirements.lock`),
bash. Время: ~1 минута.

```bash
bash code/run.sh
```

Скрипт: (1) прогоняет прогноз по трём сценариям → `data/curated/
forecast_v2026_1.csv` и `web/public/data/forecast.json`; (2) бэктест
2009→2019 (гейты: национальный итог ±2% — факт −0,41%; MAPE лучше наивной
экстраполяции — 2,39% против 4,13%); (3) чувствительность (±0,1 СКР,
±1 год ОПЖ, ±50% миграции); (4) инварианты; (5) сверку с
`checks/expected_results.json` (11 метрик, допуск 1 человек / 0,02 п.п.).

## Валидация

Отчёт — `docs/notes/validation.md`: бэктест, сверка с WPP 2024 и WCDE
(таблица 2030/2040/2050/2075, объяснение расхождений >5%), эластичности.
Гейты WP-F4/WP-F5 из TASK_SPEC — пройдены; статус закреплён pytest-тестами
репозитория и `checks/tests/test_invariants.py` пакета.

## Состав

`etl/forecast/` — код модели (идентичен репозиторию проекта; `etl/common.py`
здесь — мини-шим путей); `data/curated/` — входы (структуры 2026, переписи
2009/2019, ASFR, таблицы дожития, миграционная матрица);
`data/raw/wpp2024|wcde/` — контрольные прогнозы; `sources/registry.csv` —
реестр первоисточников (18 позиций, sha256, лицензии, даты обращения);
`docs/notes/` — валидация. Директории `sources/raw/` нет: все первоисточники
завендорены в репозитории проекта и перечислены в реестре — пакет содержит
их производные (curated) со ссылками.
