# Changelog пакета forecast

## v1.0.1 — 2026-07-11 (прогноз v2026.1, результаты не изменились)

Исправление входных данных по итогам независимого агентного аудита этапа 4:
в `data/curated/age2009.csv`/`age2019.csv` устранена коллизия territory_id —
внутригородской Октябрьский район Минска ошибочно получал id Октябрьского
района Гомельской области (158 293 чел. с oblast=BY-HM под id r-akciabrski).
На прогноз коллизия не влияла (модель читает только областные строки):
все контрольные метрики, бэктест (−0,41%, MAPE 2,39/4,13) и итоговые файлы
прогноза идентичны v1.0.0. Также исправлена неиспользуемая константа
CURATED в `etl/common.py` пакета.

## v1.0.0 — 2026-07-11 (прогноз v2026.1)

Первый релиз, этап MVP: страна, 6 областей, Минск; горизонт 2026–2075,
шаг 5 лет; сценарии base (по допущениям UN WPP 2024 medium), optimistic,
negative; 80% интервал для базового (перенос PI WPP). Гейты пройдены:
калибровка base-2050 = +1,0% к медиане WPP (порог ±3%); бэктест 2009→2019 —
национальный итог −0,41% (порог ±2%), MAPE 2,39% против 4,13% у наивной
экстраполяции. 11 контрольных метрик.

Перед публикацией пакет прошёл независимый агентный аудит: все метрики
воспроизведены байт-в-байт, детерминизм подтверждён повторным прогоном;
по итогам уточнена формулировка стресса допущения внутренней миграции
в AGENT.md (композиционный дрейф национального итога до ±0,2%).
