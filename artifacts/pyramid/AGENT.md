# AGENT.md — инструкция для ИИ-агента

Пакет `pyramid` v1.0.0: возрастно-половые структуры Беларуси
1959–2075 (17 групп × пол, 128 кадров с типами достоверности).

## Быстрый старт

```bash
bash code/run.sh
```

Python ≥ 3.10, stdlib. Шаги: сборка pyramids.json из завендоренного
сырья → инварианты → кросс-чек WPP → сверка 14 метрик. Любое
расхождение — ненулевой код выхода.

## Три обязательных задания

1. **Сверка опорных кадров с первоисточниками.** Открой вендоренные
   HTML (data/raw/pyramid/census_*.html) и независимо распарси строку
   «Всего» каждой переписи; сравни с expected_results (census_1959 …
   census_1989). Для 2009/2019 сложи data/curated/age2009/2019.csv по
   областям и сверь с 9 503 807 / 9 413 446.
2. **Байт-воспроизводимость.** Запусти `python -m etl.pyramids`
   дважды; sha256 web/public/data/pyramids.json обязан совпасть.
   Затем сверь с checks/checksums.sha256.
3. **Стресс-тест формы 2075.** Сравни доли 65+ и 0-4 в
   2075:base / 2075:optimistic / 2075:negative и 2075:base:adjusted;
   покажи, что «гриб» (масса наверху) сохраняется во ВСЕХ сценариях,
   а сценарии различаются размером основания; сравни с WPP-medium
   (checks/wpp_crosscheck.py) - форма согласована, уровень -4,3%.

## Правила интерпретации

- Тип кадра обязателен в любом выводе: census/estimate точны,
  interpolated - визуальная интерполяция, model - сценарий (не
  предсказание).
- Кадр 1999 - оценка (см. LIMITATIONS п.2).
- unknown в бары не входит, в итоги входит.
- Числа аннотаций A3/A4 за 2019 - из оценки на 01.01.2019
  (LIMITATIONS п.6).
