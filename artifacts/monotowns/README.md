# Моногорода и градообразующие предприятия: проверяемый пакет (INF-06, v1.0.0)

Пакет воспроизводит все числа исследования
«[Моногорода](https://by-population-maps.vercel.app/research/monotowns)»:
реестр 49 пар «город — предприятие», типологию по отраслям, полосу
риска и matched-comparison траекторий населения с «типовыми» городами
того же размера.

## Воспроизведение одной командой

```bash
bash code/run.sh
```

Требования: Python ≥ 3.10, только стандартная библиотека (~2 с).

## Данные

Реестр — ручная курация по открытым источникам (сайты предприятий,
отраслевая пресса, годовые отчёты, санкционные списки EU/US/UK/Canada):
`data/raw/monotowns/registry.json` (полный, с санкциями и построчными
источниками) и `data/curated/monotowns.csv` (плоский, с URL). Ряды
городов — база проекта (1959–2026).

## Главный результат

- **Чем сильнее зависимость, тем хуже** (ассоциация, не причинность):
  среди сопоставимых по размеру моногородов те, что сильно зависят от
  одного завода, к 2026 отстали от типовых городов того же размера на
  медиану **−7 п.п.**, при средней зависимости — **+3 п.п.**
- **15 крупнейших моногородов сравнивать не с чем**: Солигорск, Жодино,
  Новополоцк, Жлобин, Бобруйск и др. сами и есть крупные города Беларуси
  вне облцентров — «типового» города их размера нет; их показываем
  индивидуально (Жодино вырос на БелАЗе, старые промцентры просели).
- **Полоса риска — проспективная**: 10 городов высокого риска (сильная
  зависимость + санкции) наиболее уязвимы при остановке предприятия.
- **16 из 46** предприятий — под адресными санкциями (с датами).

## Структура пакета

- `data/raw/monotowns/registry.json` — реестр 49 пар с санкциями и
  источниками; `data/curated/monotowns.csv` — плоский с URL;
- `etl/monotowns.py` — типология, полоса риска, matched-comparison (stdlib);
- `web/public/data/monotowns.json` — итог лендинга;
- `params/assumptions.yaml` — допущения; `checks/` — 10 метрик.

Методика, ограничения (причинность не заявляется), инструкции —
LIMITATIONS.md, PROVENANCE.md, AGENT.md.
