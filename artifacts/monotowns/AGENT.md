# Инструкция для ИИ-агента: аудит пакета INF-06 «Моногорода»

Ты работаешь с проверяемым пакетом исследования. Всё, что заявлено на
лендинге, обязано воспроизводиться отсюда одной командой.

## Быстрый старт

```bash
bash code/run.sh   # Python >= 3.10, стандартная библиотека, ~2 с
```

Ожидаемый финал: `Все 10 контрольных метрик воспроизведены в допусках.`

## Что проверять

1. **Сорсинг**: у каждой из 49 записей `data/curated/monotowns.csv`
   есть URL-источники и, где заявлены, санкции с юрисдикцией и датой.
   Выборочно открой источники (при доступе в сеть) и сверь факты.
2. **Полоса риска**: пересчитай из зависимости (high/medium/low → 2/1/0)
   + санкции (есть → +1); 3=высокий, 2=повышенный, 1=умеренный, 0=низкий.
3. **Matched comparison своим кодом**: для города возьми 8 ближайших по
   населению-1989 не-моногородов и не-облцентров, посчитай медианный
   индекс к 1989 и разрыв — сверь с monotowns.json.
4. **Индекс к 1989**: сверь с рядами города в data.json.
5. **Детерминизм**: два прогона — байт-в-байт одинаковый monotowns.json.

## Ловушки

- **Причинность НЕ заявляется** — градиент по зависимости описательный
  (АССОЦИАЦИЯ); не «чини» его в причинный (LIMITATIONS §1).
- **Калипер обязателен**: разрыв считается только для города с ≥ 4
  сопоставимыми по размеру контролями (фактор ~2); у 15 крупнейших
  gap=None (нет города их размера вне облцентров) — это ЧЕСТНО, не
  дефект; без калипера прежний код матчил города 5–25× разного размера.
- **«Высокий риск» — проспективный** (уязвимость при остановке завода),
  а не свершившийся спад.
- **Санкции — снимок 2026-07-12**: не считай устаревшими; поле «на дату».
- Облцентры (Гомель/Гродно/Могилёв) исключены из реестра — не моногорода.
