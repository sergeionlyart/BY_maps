# Инструкция для ИИ-агента: аудит пакета INF-03 «Зарплата × динамика»

Ты работаешь с проверяемым пакетом исследования. Всё, что заявлено на
лендинге, обязано воспроизводиться отсюда одной командой.

## Быстрый старт

```bash
bash code/run.sh   # Python >= 3.10, стандартная библиотека, ~5 с
```

Ожидаемый финал: `Все 11 контрольных метрик воспроизведены в допусках.`

## Что проверять

1. **Сырьё**: sha256 файлов `data/raw/wages/*.json` против
   `sources/registry.csv`; при доступе в сеть — повтори запрос к API
   (эндпойнт и тело в реестре; браузерный User-Agent обязателен, лимит
   10 лет на запрос) и сравни значения.
2. **Деноминация-2016**: значения 2010–2015 в сырье — старые рубли
   (страна-2010 = 1 217 313); код делит на 10 000. Проверь, что
   дифференциалы от этого не зависят (отношение район/Минск одного года).
3. **Детерминизм**: два прогона — байт-в-байт одинаковый wages.json.
4. **Регрессия своим кодом**: пересчитай OLS (118 точек, 2 регрессора)
   любым инструментом — beta по ln(дифференциала) должна совпасть с
   54,333 до третьего знака.

## Стресс-тесты допущений (params/assumptions.yaml)

Перед перезапуском с правками кода удаляй байткод-кэш
(`rm -rf etl/__pycache__`).

- **Состав пригородов** (`MINSK_SUBURBS`): расширь до 6–8 районов
  агломерации — эластичность продолжит падать к ~2–2,5 п.п. на +10%,
  оставаясь значимой; сузь до одного Минского — вырастет.
- **Окна** (`WINDOW`, `WAGE_YEARS`): любые окна длиной 8–15 лет внутри
  2009–2025 дают положительный значимый коэффициент.
- **Контроль**: замена ln(население центра) на ln(население района)
  мало меняет главный коэффициент (проверь).

## Ловушки

- `etl/census_age.py` включён ради словаря имён районов
  (RAION_RU2ID); его функции parse()/main() ссылаются на OLAP-дампы,
  которых в пакете нет, — не вызывай их, это не дефект.
- Имена районов в сырье — прилагательные без «район» («Оршанский»),
  «Берёзовский» с ё; матчинг нормализует ё→е.
- Связь корреляционная: не интерпретируй beta как причинный эффект —
  см. LIMITATIONS.md.
- «Не распределено по районам Минской области» (с 2024) — служебная
  строка, исключается осознанно.
