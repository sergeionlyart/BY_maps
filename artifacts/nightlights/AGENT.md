# Инструкция для ИИ-агента: аудит пакета INF-08 «Ночные огни»

Ты работаешь с проверяемым пакетом исследования. Всё, что заявлено на
лендинге, обязано воспроизводиться отсюда одной командой.

## Быстрый старт

```bash
bash code/run.sh   # Python >= 3.10, стандартная библиотека, ~2 с
```

Ожидаемый финал: `Все N контрольных метрик воспроизведены в допусках.`

## Что проверять

1. **Целостность**: sha256 `data/raw/nightlights/zonal_light.csv` против
   `sources/registry.csv`; там же — URL и официальный sha256 каждого
   исходного композита WorldPop (сами растры ~30 МБ/год не вендорятся;
   при доступе в сеть скачай по URL и сверь sha256).
2. **Доли, не уровни**: индекс строится на ДОЛЯХ зоны в национальной
   светимости и населении — это гасит версионный скачок VNL 2.1→2.2.
   Не сравнивай абсолютные радиометрии между годами.
3. **Индекс своим кодом**: для 2-3 крупных районов пересчитай долю
   света, лог-линейный тренд 2015–2019, отношение к 2022–2023; индекс =
   ln(отн. доли света) − ln(отн. доли населения). Минск ≈ 0,
   индустриальные (Жодино/Борисов/Гомельский) < 0.
4. **Детерминизм**: два прогона — байт-в-байт одинаковый nightlights.json.

## Ловушки

- `etl/nightlights_extract.py` — документация шага извлечения зон из
  100-м композитов (требует rasterio и сами растры); в run.sh НЕ
  вызывается — расчёт идёт от `zonal_light.csv` на stdlib.
- **Свет ≠ население**: индекс — маркер, не оценка численности; не
  интерпретируй отрицательное расхождение как доказанный отток
  (LIMITATIONS §1). Индустриальное расхождение равно объясняется
  деиндустриализацией.
- **Малые районы шумны** — выводы по крупным; не «чини» шум малых.
- Минск (BY-HM) — отдельная зона из adm1: в adm2 он «дыра» Минского
  района. Минск и Минский район — два ярчайших источника, периодически
  меняются местами (город плотный, район большой) — это не дефект.
- Источник — ПОДЛИННЫЙ EOG VNL (average_masked), а не смоделированный
  «NPP-VIIRS-like» (тот отвергнут за нефизичную нестабильность —
  LIMITATIONS §9).
