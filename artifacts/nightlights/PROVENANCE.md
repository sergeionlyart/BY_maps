# Происхождение данных (INF-08, v1.0.0)

```
EOG VNL 2.1/2.2 «average_masked» (Colorado School of Mines / Payne
Institute) — подлинный годовой композит VIIRS DNB
        │  обработка WorldPop: вариант fvf (без газовых факелов и
        │  вулканов), ресемпл 100 м, обрезка по Беларуси, 2015-2023
        ▼  https://data.worldpop.org/GIS/Covariates/Global_2015_2030/
        │  BLR/VIIRS/v1/fvf/blr_viirs_fvf_<год>_100m_v1.tif
        │  (URL + официальный sha256 - sources/registry.csv;
        │   растры ~30 МБ/год В РЕПОЗИТОРИЙ НЕ ВЕНДОРЯТСЯ)
        ▼  etl/nightlights_extract.py (rasterio, однократно):
        │   растеризация 118 районов (adm2) + Минск (adm1) на сетку
        │   композита; сумма радиометрии > 1 нВт/см²/ср по зоне
        ▼
  data/raw/nightlights/zonal_light.csv  (119 зон x 9 лет)
        │
        ▼  etl/nightlights.py (стандартная библиотека):
  доля зоны в нац. светимости и в нац. населении (поле `pop`;
  район включает свой город) -> лог-линейный тренд долей 2015-2019 ->
  отношение к 2022-2023 -> индекс расхождения =
  ln(доля света_ф/тренд) - ln(доля населения_ф/тренд)
        │  ряды населения: web/public/data/data.json
        ▼
  web/public/data/nightlights.json
```

Все шаги от `zonal_light.csv` детерминированы и идут на стандартной
библиотеке; rasterio нужен только для однократного извлечения зон
(документируется etl/nightlights_extract.py). URL и sha256 исходных
композитов, а также sha256 зональной суммы зафиксированы в
`sources/registry.csv`.

Исходно рассматривался смоделированный «NPP-VIIRS-like» V2 (Harvard
Dataverse) — отвергнут: его годовые слои по Беларуси нефизично
нестабильны (страновая сумма 294→143→89 за 2022-2024, районы гасли в
ноль). Заменён подлинным EOG VNL, стабильность которого проверена
загрузкой (доля Минска CV ≈ 0,08).
