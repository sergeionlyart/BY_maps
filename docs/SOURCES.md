## WP-F1: данные для прогноза (этап 2)

Полный машиночитаемый реестр с URL, лицензиями, датами обращения и sha256 —
[`data/raw/registry_wpf1.csv`](../data/raw/registry_wpf1.csv). Ключевое:

| Набор | Источник | Что внутри |
|---|---|---|
| `data/curated/age2009.csv`, `age2019.csv` | OLAP-куб F201N, census.belstat.gov.by (гостевой доступ) | обе переписи: 118 районов + 12 городов обл. подчинения × 5-летние группы × пол × город/село; суммы сходятся с официальными итогами **до человека** |
| `data/curated/age_current.csv` | открытый API dataportal.belstat.gov.by (индикатор 10101100003) | на 1 января 2019–2026: страна/области/Минск × 5-летние группы × пол × местность |
| `data/curated/migration_internal.csv` | OLAP-куб F602 | межобластная матрица миграции × возрастные группы, переписи 2009/2019 |
| `data/curated/fertility.csv` | HFD (открытое зеркало OWID) | ASFR Беларуси 1964–2018, однолетние возрасты, по очередности рождения |
| `data/curated/mortality.csv` | HMD (открытое зеркало OWID) | полные таблицы дожития 1959–2018, однолетние возрасты |
| `data/raw/wpp2024/*` | UN WPP 2024 (CC BY 3.0 IGO) | контрольный прогноз: Medium/High/Low по однолетним возрастам 2024–2100 + все 18 вариантов итогов |
| `data/raw/wcde/*` | Wittgenstein Centre v3 (CC BY 4.0) | контрольный прогноз: SSP1/SSP2/SSP3, итоги и половозрастные структуры |

Выгрузка OLAP невозможна обычным HTTP-клиентом (сервер принимает изменённые
запросы только из браузерного контекста) — процедура: `etl/olap_receiver.py`
(локальный приёмник) + fetch-вызовы из браузерной вкладки; сырые ответы
завендорены. Повторная выгрузка dataportal: `python -m etl.fetch_age_current`.

## Проверочные константы (в тестах)

Официальные доли городского населения по переписям — 1999: 69,3%, 2009: 74,5%,
2019: 77,6% (Белстат, итоги переписей) — используются как эталон для проверки
вычисленных сумм по городам (допуск 1,5 п.п.).
